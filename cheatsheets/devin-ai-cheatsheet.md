# Devin AI 에이전트 치트시트

> Cognition Labs의 자율 AI 소프트웨어 엔지니어 Devin — 한 페이지 요약

## 기본 정보

| 항목 | 내용 |
|------|------|
| 개발사 | Cognition Labs |
| 유형 | 클라우드 호스팅 자율 코딩 에이전트 |
| 실행 환경 | 웹 브라우저 (로컬 설치 없음) |
| 인터페이스 | Slack 연동 + 웹 IDE |
| 최신 버전 | Devin 2.0 |

## 요금제

| 플랜 | 가격 | 주요 기능 |
|------|------|----------|
| Core | $20/월 | 기본 기능, ACU당 $2.25 과금 |
| Team | $500/월 | 무제한 동시 세션, 팀 관리 |
| Enterprise | 커스텀 | SSO, SLA, 전용 지원 |

> **ACU(Agent Compute Unit)** — Devin이 작업에 사용하는 컴퓨팅 단위. 복잡한 작업일수록 ACU 소비가 커요.

## 핵심 기능

| 기능 | 설명 |
|------|------|
| 자율 실행 | 태스크를 할당하면 독립적으로 코딩, 테스트, 디버깅 |
| 환경 구축 | 프로젝트 셋업, 의존성 설치, 빌드 자동 처리 |
| Slack 통합 | Slack에서 직접 태스크 할당하고 결과 확인 |
| 플래너 | 작업 계획을 먼저 세우고 단계별로 실행 |
| 브라우저 내장 | 문서 검색, API 참조를 직접 브라우저로 확인 |
| 스냅샷 | 작업 중간 상태를 저장하고 되돌리기 가능 |
| PR 생성 | 완료된 작업을 GitHub PR로 자동 생성 |
| 멀티 레포 | 여러 저장소를 동시에 작업할 수 있음 |

## 잘 맞는 작업

```
✅ 반복적인 CRUD 구현        — 비슷한 패턴의 엔드포인트 여러 개
✅ 버그 재현 + 수정          — 이슈 링크만 주면 분석 후 패치
✅ 마이그레이션 작업          — 라이브러리 버전 업, API 변경 적용
✅ 테스트 코드 추가          — 기존 코드 기반으로 테스트 스위트 생성
✅ 문서화                   — README, API 문서, 타입 정의 보완
✅ 의존성 업데이트            — 보안 패치, 버전 충돌 해결
```

## 주의가 필요한 작업

```
⚠️ 복잡한 아키텍처 설계      — 사람의 판단이 필요한 설계 결정
⚠️ 대규모 리팩토링           — 파일 간 의존성이 복잡한 변경
⚠️ 프로덕션 배포             — 자율 실행 범위 밖으로 두는 게 안전
⚠️ 보안 관련 코드            — 인증, 암호화 로직은 반드시 사람이 검토
⚠️ 성능 최적화              — 프로파일링 결과 해석에 한계
```

## Slack 사용법

```
# Slack에서 Devin에게 태스크 할당
@devin 이 이슈 수정해줘 https://github.com/org/repo/issues/42

# 작업 범위 지정
@devin repo/src/api 폴더에 pagination 추가해줘

# 진행 상황 확인
@devin status

# 작업 중단
@devin stop
```

## Knowledge 설정

```
📁 프로젝트 루트에 devin.md 파일 추가
  → 프로젝트 구조, 코딩 컨벤션, 테스트 방법 등 작성
  → Devin이 작업 시작 전에 참조

📁 팀 Knowledge (Team 플랜)
  → 공통 가이드라인을 팀 전체 세션에 적용
```

```markdown
<!-- devin.md 예시 -->
# 프로젝트 가이드

## 기술 스택
- Next.js 15 + TypeScript
- Prisma ORM + PostgreSQL

## 코딩 규칙
- 모든 함수에 JSDoc 주석
- 에러는 커스텀 에러 클래스 사용
- 테스트: vitest + testing-library

## 브랜치 규칙
- feature/* 브랜치에서 작업
- 커밋 메시지: conventional commits
```

## 다른 도구와 비교

| 항목 | Devin | Claude Code | Cursor |
|------|-------|-------------|--------|
| 실행 환경 | 클라우드 (웹) | 로컬 (터미널) | 로컬 (IDE) |
| 자율성 | 높음 (독립 실행) | 중간 (대화형) | 낮음 (실시간 협업) |
| 환경 구축 | 자동 (VM 제공) | 수동 (로컬 환경) | 수동 (로컬 환경) |
| 오프라인 | 불가 | 가능 (로컬 모델) | 불가 |
| 비용 모델 | ACU 기반 종량제 | 구독 + 토큰 | 구독제 |
| 코드 프라이버시 | 클라우드 전송 | 로컬 유지 가능 | 클라우드 전송 |
| 적합한 작업 | 독립 태스크 위임 | 복잡한 로직/리팩토링 | 실시간 코드 작성 |

## 팀 도입 시 고려사항

| 체크 항목 | 설명 |
|----------|------|
| 코드 보안 정책 | 클라우드로 코드가 전송되므로 보안 정책 확인 필수 |
| ACU 예산 관리 | 복잡한 작업에서 ACU 소비가 급증할 수 있음 |
| 결과물 리뷰 프로세스 | 자율 생성된 PR은 반드시 사람이 리뷰 |
| 기존 CI/CD 연동 | Devin PR → 기존 파이프라인으로 검증 |
| fallback 계획 | Devin이 실패한 태스크는 사람이 직접 처리하는 루트 |

## 효과적인 태스크 할당 팁

```
1. 구체적으로 지시해요
   ❌ "이 코드 개선해줘"
   ✅ "src/api/users.ts의 getUser 함수에 캐싱 레이어 추가해줘.
       Redis 사용, TTL 5분, 캐시 미스 시 DB 조회"

2. 참조 파일을 명시해요
   "src/api/posts.ts의 getPost 함수를 참고해서 동일한 패턴으로"

3. 테스트 기준을 알려줘요
   "vitest로 단위 테스트 포함, 캐시 히트/미스 케이스 모두"

4. 완료 조건을 명확히 해요
   "모든 기존 테스트 통과 + 새 테스트 추가 + lint 에러 없음"
```

## 주요 한계

| 한계 | 대응 방법 |
|------|----------|
| 복잡한 작업에서 반복 실패 | 태스크를 더 작은 단위로 분할 |
| 컨텍스트 유실 | devin.md에 핵심 컨텍스트 정리 |
| 빌드/테스트 환경 차이 | 프로젝트 셋업 스크립트 정비 |
| 예상보다 높은 ACU 비용 | 작은 작업부터 시작해서 비용 패턴 파악 |
| 보안 민감 코드 변경 | Devin 작업 범위에서 제외, 사람이 직접 |
